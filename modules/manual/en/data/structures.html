Structures

About directories and files

Loggix's file structure is shown below. (Click linked icons for more details.)

  • admin (System admin's directory)
    • delete.php
    • drafts.php
    • edit.php
    • index.php
    • info.php
    • login.php (login file)
      Access to this file to log in.
    • modify.php
    • post.php
    • tags.php
    • users.php
    • write.php
  • data (Data directory)
      SQLite database and uploaded files and images are stocked in here.
    • resources (Directory for mages and files)
    • loggix.sqlite3.db (SQLite3 database file)
    • loggix.sqlite.db (SQLite2 database file)
  • lang (Language resorce directory)
    • english.lang.php
    • japanese.lang.php
  • lib (Library file directory)
      Loggix core libraries and thirdparty libraries such as PEAR is placed in here. See also "Loggix API" for more info.
    • Loggix (Loggix class files)
      • Module.php (module class)
        Extend this class to make your own module class.
      • Module (Extended Module directory)
          Extended modules are placed in here. All classes will be autoroaded when a Loggix application is created.
        • Calendar.php
        • Comment.php
        • Rss.php
        • Trackback.php
      • View.php (View class)
        Loggix's template engine. It's really simple and tiny but good and fast enough to handle a view of small size website.
      • View (Directory for subclasses of Loggix_View)
        • Helper.php (View helper class)
      • doc
      • Application.php (Application class)
        This class will be used to create a Loggix applicaion.
      • Core.php (Core class of Loggix)
        Loggix core handles database controll.
      • Exception.php (Exception class)
      • Expander.php (Loggix_Expander class)
      • Plugin.php (Plugin class)
        This class adds plugin architectures to Loggix application.
      • Session.php (Session class)
      • Utility.php (Utility class)
    • php (Directory for thirdparty libraries)
        You can load files here with include or require from anywhere in Loggix applicatoin.
      • PEAR
      • PEAR.php
  • modules (Directory for modules)
    • calendar
    • comment
    • downloads
    • helloworld
    • manual
    • rss
    • trackback
    • index.html
  • plugins (Directory for plugins)
      Please note that all plugins here will be autoloaded.
    • off (Move plugin here to make it off)
    • captcha.php
    • captcha
    • denyCommentByIp.php
    • denyCommentByName.php
    • denyTrackbackWithNoReference.php
    • displayEntryDeletedMessage.php
    • hilightKeyword.php
    • markdown.php
    • showSamplesMenu.php
    • showTrackbackStatus.php
  • theme (Directory for theme files)
      Loggix theme files are standard (X)HTML and CSS files.
    • admin (admin theme directory)
    • css (stylesheet directory)
    • data (data theme directory)
    • errors (Error message theme directory)
    • images (Shared image files)
    • js (JavaScript files directory)
    • archives.html
      Arvhive view file
    • archives-by-date.html
      Archive by date view file.
    • archives-by-tags.html
      Arvhive by tags view file.
    • base.html (Base (X)HTML view file.)
      Base (X)HTML view file.
    • default.html
      This will be used when no data is available.
    • navigation.html
      navigation view file.
    • parmalink.html
      Parmalink view file.
    • search-result.html
      Search result view file.
    • smiley-button.html
      Smiley button view file.
    • tag-result.html
      Tag search result view file.
  • index.php (Front controller)
    Vistors will access this file. This script will handle all requests and controll database engine.

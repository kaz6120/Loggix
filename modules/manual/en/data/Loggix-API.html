Loggix Core Classes

Loggix Core classes controll the Loggix's database and file handlings and located in /lib/Loggix/ directory. All packages are named like : "Loggix_*".

    • Loggix
      • Application.php

      • Utility.php
      • View.php

Loggix Module Classes

Loggix Module Classes are built for expand Loggix's functions which is located in /lib/Loggix/Module/ directory. All these class modules will be auto-loaded when you launch Loggix application. You can add your original or customized module to the directory.

  • lib
    • Loggix

      • Module

        • Trackback.php

How to extend Loggix_Module

When you develpe Loggix module exntends Loggix_Module, you should follow rules below.

  1. First of all, you are recomended to use Loggix_Module class when you create your own module. You can access all of the core method through it.
  2. You have to use prefix to your class name. (see: Example)
  3. Loggix's default core modules are all named with prefix: Loggix_Module_, this means "This Module extends Loggix_Module". (All the thirdparty modules included in package are not renamed.)
  4. You should use class to make your own module to make it package.
  5. You should include your creator code name in the first variable group like this:
    $module['LM']['foo']
    In this example you can see Loggix Project's creator code "LM" ( stands for Loggix Module) is used in the first variable group. Only Loggix Project can use this creator code, so you have to name your own creator code. Please don't forget to make it short. When you want to separate by words, use underscore.
  6. Follow Coding Standards.

Example

Here's an example class, "Example".

  • lib
    • Loggix
      • Module
        • Example.php (module class file)

Name your class like this:

<?php
class Loggix_Module_Example extends Loggix_Module
{
    public function getFoo()
    {
        return $module['LM']['foo'] = 'Foo';
    }
}

Where can I put my module anyway?

You should add your own module to "/module" directory.

  • modules
    • example(lower-case without prefix)

All modules will be auto-loaded automatically.

For class member variables that are declared with the "private" or "protected" construct, the first character of the function name must be a single underscore. This is the only acceptable usage of an underscore in a function name. Member variables declared "public" may never start with an underscore.

Like function names (see section 3.3, above) variable names must always start with a lowercase letter and follow the "camelCaps" capitalization convention.
